Ruby Classes vs. JavaScript Constructor functions

October 25, 2015

What is a Ruby class?

Ruby Classes was explained in the previous blog. This link will bring you to it. It gives a in-depth explanation of what Ruby classes are and what they do. Ruby classes are pretty much like a mother bird protecting their babies. It should contain all the methods for that classes category. An example would be a board class. The board class should contain all methods involving the creation of a board or anything of the sort.

class Board
  attr_reader :size
  def initialize(size)
    @size = size
  end
end
new = Board.new(16)
new.size  => 16

What is a JavaScript constructor function?

Constructor functions are JavaScript objects. It creates a new object and can be created by using "new" like in the example below. The "new" sets the "this" binded to the object. 

function theObject() {
  this.object = "Object";
}
newObj =  new theObject();

The differences

JavaScript uses literal notation or constructor functions to create a new object. There are a lot more syntax needed in JavaScript than in Ruby as well. JavaScript requires semicolons and parenthesis but Ruby doesn't require as much. 

JavaScript
function myObject() {
  this.object = "hello";
}

Ruby
Class ObjectOne
  def initialize(name)
    @name = name
  end
end
